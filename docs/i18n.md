# i18n Guide

## 구조

```
src/
  lib/
    i18n.ts              # i18next 초기화 설정
  locales/
    ko/
      common.json        # 공통 번역 (네비게이션, 버튼 등)
      auth.json           # features/auth 번역
      game.json           # features/game 번역
      lobby.json          # features/lobby 번역
      profile.json        # features/profile 번역
    en/
      common.json
      auth.json
      game.json
      lobby.json
      profile.json
```

## 네임스페이스 규칙

- feature별로 네임스페이스를 분리 (auth, game, lobby, profile)
- 2개 이상 feature에서 공유하는 키는 `common` 네임스페이스에 위치
- 기본 네임스페이스: `common`
- 기본 언어: `ko`

## 번역 키 네이밍

- camelCase 사용: `loginButton`, `emailPlaceholder`
- 중첩 구조 활용: `{ "form": { "email": "이메일", "password": "비밀번호" } }`
- 페이지/컴포넌트 단위로 그룹핑

## 사용법

```tsx
// 기본 (common 네임스페이스)
const { t } = useTranslation()
t("confirm") // common.json의 confirm

// feature 네임스페이스
const { t } = useTranslation("auth")
t("loginButton") // auth.json의 loginButton

// 중첩 키
t("form.email") // { "form": { "email": "이메일" } }

// 보간
t("welcome", { name: "홍길동" }) // "환영합니다, {{name}}님"
```

## 주의사항

- 하드코딩된 한국어 문자열 금지 — 반드시 t() 사용
- 새 feature 추가 시 해당 네임스페이스 JSON 파일도 함께 생성
- 키를 삭제할 때는 모든 언어 파일에서 함께 삭제
